### 2-3. 勝者判定（★タイブレーク順位・確定値・変更禁止）

```
shape > bard > sniper > sage > gunner > golem > sword > archer
```

> ⚠️ この順位はデフォルト（設問順）ではない。**結果の偏り是正のために全40320順列を総当たり探索して確定した値**。素朴な設問順だとガンナーが33%に突出する（格差8.1倍）。**勝手に変えないこと。**

```javascript
const METERS   = ['gunner','sniper','golem','sword','archer','sage','shape','bard'];
const PRIORITY = ['shape','bard','sniper','sage','gunner','golem','sword','archer'];

// ★★ R26：必ずこれを使ってスコアを初期化する ★★
// 8キーを0で初期化しないと、scores[key] が undefined になり、
// 全ての比較が false になって best が PRIORITY[0] = 'shape' のまま返る。
// = 全員シェイプシフターになる。しかも無言で。絶対にやらない。
function createScores() {
  return METERS.reduce((acc, k) => (acc[k] = 0, acc), {});
}

function decideJob(scores) {
  // ★防御：8キーが揃っていなければ、静かに間違うのではなく、うるさく落とす
  for (const k of METERS) {
    if (typeof scores[k] !== 'number') {
      throw new Error(`scores["${k}"] が未初期化です。createScores() を使ってください。`);
    }
  }
  let best = PRIORITY[0];
  for (const key of PRIORITY) {
    if (scores[key] > scores[best]) best = key;
    // 同点は "先に来た方(=既存best)" を維持 → PRIORITY順で自動的に先勝ち
  }
  return best;
}
```

### 5-2. ジョブ判別導線（★合言葉方式・FIX）

**方式**：結果画面／レポートのLINE導線押下時に、**プレーンなジョブ名**をクリップボードにコピー → LINE友だち追加URLへ遷移 → ユーザーがトークに貼り付け（または手打ち）して送信 → **キーワード自動応答**で 3つの地雷リスク＋最初のミッションを返す。

> ⚠️ **`CONFIG` の定義は 0-4a にある。ここには再掲しない**（同じ値を2箇所に書かない）。

```javascript
async function goLine(jobKey, location) {   // location: 'result' | 'report'
  const name = JOBS[jobKey].name;            // 例: '連射のガンナー'（★絵文字なし・プレーン）
  gtag('event', 'line_cta_click', {
    job_type: jobKey,
    location: location,                                        // ★R27
    variant: location === 'result' ? 'text_link' : 'button',   // ★R27
  });
  try {
    await navigator.clipboard.writeText(name);
    showToast(`「${name}」をコピーした。LINEに貼って送るだけ。`);
  } catch (e) {
    showFallback(name);                      // ← コピー失敗時：選択可能なテキストで表示
  }
  window.location.href = CONFIG.LINE_URL;
}
```

**厳守事項**
- コピーする文字列は **絵文字なしのジョブ名のみ**（例：`連射のガンナー`）。絵文字は**UI表示のみ**。異体字セレクタでキーワード完全一致を外すため。
- Clipboard API は **HTTPS必須 ＋ クリックハンドラ内で同期的に呼ぶ**こと（GitHub Pagesはhttps）。
- **必ず try/catch。** 失敗時は `<input readonly value="連射のガンナー">` 等で**選択・コピーできる状態**にして表示する。コピー成功を前提にしない。
- **PC（またはコピー不可時）**：LINE導線の近くに **QR画像**＋「トークにこう送ってね ▶ **連射のガンナー**」を**大きく可読に**表示。
- ⚠️ **クリップボードはデバイス単位。** PCでコピーしても**スマホのクリップボードには渡らない**。PCユーザーはQRを読んでスマホでLINEを開くので、**手で打てる状態が必須**。またスマホでも、ブラウザ→LINEアプリ遷移でクリップボードが保持されるかは**端末依存・未検証**。→ **LINEのあいさつ文で「コピー済み」と断言しない**（→ コピー設計 v3.2 §5-1）。

> 💡 v1.1候補：LINE URLスキーム `https://line.me/R/oaMessage/{ベーシックID}/?{テキスト}` を使うと入力欄にジョブ名がプリセットされる（＝タイピングゼロ）。ただし **スマホ限定**（公式マニュアル明記）かつ**友だち未追加時の挙動が未検証**のため、MVPでは採用しない。

## 6. デザイン方針

### 6-1. 共通

- **世界観**：バトルロワイヤル。「降下前のロビー／戦場」の空気。ダークテーマ必須（濃紺〜黒＋アクセントカラー）。
- ★**ジョブのアクセント色（R36・厳守）**：**`theme.css` の `[data-job="…"]` 8行だけが色の定義場所**（→ 0-4c）。使う側は**必ず `var(--job-accent)`**。**HEX値をHTML・コンポーネント・JSに直書きすることを禁止する。**
  - ページのルート要素（またはカード要素）に `data-job="{jobKey}"` を付けるだけで色が切り替わる。
  - **色の実値は未確定（仮色でよい）。** 確定後に `theme.css` の8行を差し替えるだけで、全ページに反映されること。**これが満たせていない実装は不合格。**
- **UI語彙**：スタート＝「降下する」、結果＝「**JOB CONFIRMED**」。演出過多で読み込みを遅らせない。
- **プログレス表記（★R34で明確化）**：`Q3 / 5` とする。~~`残り 3 / 5`~~ は「残り3問」なのか「3問目」なのか曖昧で、実装のバグ元になる。
- ⚠️ **銃・暴力表現に寄せすぎない**。血・照準・カウントダウン等の恐怖演出はNG。「ゲームの選択画面」のトーン。
- **モバイルファースト**。指で押しやすい大きめの選択肢ボタン。
- **結果画面はスクショされる前提**：1画面に「ジョブ名・絵文字（または画像）・キャッチ・棒グラフ」が収まること。
- **棒グラフ**：横棒。結果画面＝動的・上位のみ（2-6）／レポート④＝固定値・8本（2-5）。
- **ジョブ画像**：8枚（納品済み）。ファイル名は `/assets/jobs/{jobKey}.png` で固定。**画像が無くても壊れないこと**（alt＋絵文字フォールバック）。
