## 4. 単一情報源ルール（★同じ値が2箇所にあったら仕様バグ）

| 値 | 唯一の定義場所 |
|---|---|
| **ジョブのアクセント色8色** | `theme.css` の `[data-job="…"]` 8行（→ 仕様書 §0-4c） |
| **パラメータの日本語表示名** | `PARAM_LABELS`（`report/report.js`／→ 仕様書 §0-4b） |
| **CONFIG**（`GA_ID` / `LINE_URL` / `SITE_URL` / `QR_IMG`） | `CONFIG`（`report/report.js`／→ 仕様書 §0-4a） |
| **設問・選択肢** | `QUESTIONS`（`index.html`／→ 仕様書 §3） |

**理由：この4つは後日すべて差し替える。** 2箇所以上に書かれた時点で、差し替えられなくなる。

- ❌ **色のHEXをHTML・JS・`report.css` に直書きする** → 使う側は必ず `var(--job-accent)`
- ❌ **パラメータの日本語をHTML・JSに直書きする** → 必ず `PARAM_LABELS` から引く
- `<body data-job="{jobKey}">` を付けるだけで色が切り替わる構造にすること
  - 診断本体：結果表示時に `document.body.dataset.job = jobKey`
  - レポート8枚：静的に `<body data-job="gunner">`
  - 8ジョブ図鑑：**カード単位**で `<article data-job="sniper">`（1ページに8色が並ぶ）

## 8-2. ★クリップボード実装の注意（HTTPS依存）

`navigator.clipboard.writeText()` は **HTTPS（および `localhost`）でしか動かない。**
**HTTP では例外も出さずに静かに失敗する。**

- **必ず `try / catch` で囲み、失敗時のフォールバック（ジョブ名を選択可能なテキストで表示＋「手動でコピーしてください」）を実装する。**
- ⚠️ **「コピー済み」と断定するUIを出さない**（仕様書 §5-2）。PCではクリップボードがスマホに渡らないため、嘘になる。
- ローカル（`localhost`）では必ず成功するため、**この分岐はテストで踏めない。** コードとして必ず書くこと。
