## 4-2. ★公開前の実機検証（★ローカルでは絶対に発覚しない罠）

**Claude はこの4項目を、Waka に必ずリマインドすること。** レビューで緑になっても、ここが抜けると本番で死ぬ。

| # | 検証項目 | 落ちたときに起きること |
|---|---|---|
| **1** | ★**公開ドメインで HTTPS が有効か**（🔒 が出るか）を確認してから拡散 | **`navigator.clipboard.writeText()` は HTTPS でしか動かない。HTTP では例外も出さずに静かに失敗する。** → ジョブ名がコピーされない → LINEに飛んでも貼るものがない → **リード獲得が丸ごと0。GA4を見るまで誰も気づかない。**<br>⚠️ **`localhost` は例外的にHTTPS扱いになるため、ローカルテストでは100%成功してしまう。** |
| **2** | ★**LINE内蔵ブラウザで、コピー → LINE遷移が通るか** | 診断はLINEでシェアされる＝**LINE内蔵ブラウザで開かれる。** ここの `clipboard` 挙動は未検証。**推測で判断しない。実機で1分。** |
| **3** | 4吹き出しの応答で、**プッシュ通知が1回か4回か** | 4回鳴るならブロック要因。**自分のスマホで友だち追加すれば5秒で分かる。** |
| **4** | PCで診断 → **QR＋ジョブ名が読める形で出るか** | クリップボードはPC→スマホに渡らない。フォールバックの確認。 |

**⚠️ カスタムドメイン（`batorowa.hakoniwa.link`）の証明書発行には最大24時間かかる。緑になる前に拡散しない。**

## 4-3. ★DNS作業の安全ルール（hakoniwa.link 本体を殺さない）

`hakoniwa.link` は **STUDIO** でホスティングされている。同じDNSに触るので、事故ると**本体サイトが落ちる。**

| 操作 | 可否 |
|---|---|
| CNAME `batorowa` → `liongeorges.github.io` を **追加** | ✅ |
| 既存の **Aレコード（`34.111.141.225` = STUDIO）** を編集・削除 | ❌ **絶対にしない。hakoniwa.link 本体が死ぬ** |

- **サブパス（`hakoniwa.link/bizbatorowa`）は実現不可能。** STUDIOはサブディレクトリに対応しておらず、DNSはパスを振り分けられない。**サブドメインのみ。**
- ルートドメインにCAAレコードがある場合のみ、サブドメイン側にもCAA追加が必要（通常は不要）。

  裏付けがあるのは**行動スタイルの有効性**であって、**この5問による分類の妥当性ではない。**

## 7. Waka の作業スタイル（★守ってほしい）

- **結論から。casual・対等なパートナーとして話す。** 忖度・お世辞は要らない。
- **数字は必ず実際に計算・実行して確認する。** 推定・体感で答えない。
- **外部仕様（LINE・X API・GitHub Pages等）は必ず検索して裏を取る。** 「たぶんこうです」は禁止。
- **指摘は「本当に問題か」を先に説明する。** 同意を得てから直す。
- **修正指示は番号付き・厳密。** 指定外の箇所を勝手に触らない。
- MVP段階では、**完璧さより速度と低コスト。**

## 8. Phase 3 以降：データ分析の担当

種火（身内20〜50人）を回したら、Claude が見る数字：

| KPI | イベント | 見るポイント |
|---|---|---|
| ① 完走率 | `quiz_start` → `result_view` | どの設問で落ちるか |
| ② シェア率 | `share_x_click` | `location` 別（result / report） |
| ③ **リード獲得率** | `line_cta_click` | ★**`location` で分離必須。** 結果画面のテキストリンク vs レポートの大型ボタン。**「結果画面でLINEを下げれば総リードが増える」という設計判断（§5-0）は仮説。ここで初めて検証される** |
| ④ 有料v2の需要温度 | リッチメニュー『AI伴走を希望』の押下数 | **閾値はデータを見る前に決めておく**（後から決めると必ずGOになる） |

**実測分布が理論値（shape 21.2% / gunner 13.9% / bard 13.4% / golem 12.8% / sword 12.1% / archer 11.0% / sniper 9.0% / sage 6.6%）とズレたら、設問の解釈がブレている可能性がある。**
